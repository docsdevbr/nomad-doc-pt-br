# Início rápido do Nomad

Estas instalações foram projetadas para você começar a usar o Nomad e devem
ser usadas apenas para fins experimentais.
Se você deseja instalar o Nomad em produção, consulte o
[Guia de instalação em produção](/nomad/docs/deploy/production).

## Tutoriais do Nomad

Recomendamos estes tutoriais, que provisionam um cluster Nomad para você.

- [Introdução][]: Comece a usar o Nomad aprendendo sobre agendamento,
  configuração de um cluster e implantação de um job de exemplo.
  Crie um cluster Nomad em sua máquina local ou em um ambiente de nuvem AWS,
  Azure ou Google Cloud Platform (GCP).
  Esta série de tutoriais possui um repositório de código associado para que
  você possa revisar os scripts de provisionamento do Terraform e as
  especificações do job do Nomad.
- [Configuração do cluster][]: Provisione um cluster Nomad na AWS, Azure ou GCP.
  Habilite o Consul e as listas de controle de acesso (ACLs).
  Esta série de tutoriais possui um repositório de código associado para que
  você possa revisar os scripts de provisionamento do Terraform.

## Instalação na nuvem

O repositório Nomad contém guias e scripts Terraform para provisionar um

## Instalação local

Use um dos seguintes métodos para executar um ambiente sandbox local do Nomad:

- Instale um binário

  Consulte o [Guia de instalação do Nomad][installing-binary] para instalar o
  binário da versão mais recente do Nomad.  

  Execute o Nomad com o [comando][agent-dev] `sudo nomad agent -dev`, que inicia
  o agente Nomad em modo de desenvolvimento.

  Se você estiver usando o Docker Desktop para Windows ou MacOS, consulte
  [Como conectar à minha rede host ao usar o Docker Desktop (Windows e MacOS)][faq-win-mac]
  para obter instruções sobre como vincular o Nomad a uma interface de rede sem
  loopback.

- Use Vagrant e VirtualBox

  Esta opção requer hardware AMD.

  O repositório Nomad contém um arquivo Vagrant.
  Consulte a seção [Desenvolvendo com Vagrant][vagrant] do arquivo
  [Documentação da base de código do Nomad][compile-source] para obter
  instruções.

- Compile e execute a partir do código-fonte

  Esta opção requer experiência em engenharia de software e uma estação de
  trabalho configurada para desenvolvimento em Go, Node e Docker.

  Consulte o arquivo [Documentação da base de código do Nomad][compile-source]
  do repositório Nomad para obter detalhes.
